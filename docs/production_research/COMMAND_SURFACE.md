# Command Surface Standard

Last updated: 2026-02-22

Purpose:
- keep operator command surface minimal and stable;
- separate primary entrypoints from internal implementation scripts;
- reduce cognitive load and accidental misuse.

## 1) Primary Commands (Human-facing)

Use these first:

1. daily research track:
```bash
bash scripts/ops_entry.sh daily
```
2. fast status refresh:
```bash
bash scripts/ops_entry.sh status
```
3. fast research screen (non-official, isolated outputs):
```bash
bash scripts/ops_entry.sh fast --dry-run
# then real run:
# bash scripts/ops_entry.sh fast
```
4. official workstation run:
```bash
bash scripts/ops_entry.sh official ...
```
5. factor factory batch:
```bash
bash scripts/ops_entry.sh factory --dry-run --jobs 4
# bash scripts/ops_entry.sh factory --jobs 4
```
6. command-surface drift check:
```bash
bash scripts/ops_entry.sh check
```
7. safe cleanup preview / apply:
```bash
bash scripts/ops_entry.sh cleanup
# apply deletion explicitly:
# bash scripts/ops_entry.sh cleanup --apply
```
8. hygiene routine (recommended before/after doc-heavy edits):
```bash
bash scripts/ops_entry.sh hygiene
```

## 2) Internal Commands (Automation-facing)

These remain valid but are not first-choice manual entrypoints:

- `scripts/daily_research_run.sh`
- `scripts/generate_daily_research_brief.py`
- `scripts/workstation_official_run.sh`

Reason:
- they are implementation details behind `ops_entry.sh`;
- direct usage is allowed for compatibility and debugging.

## 3) Execution/Audit Constraints

1. Any command generated by `next_run_plan` must remain compatible with `scripts/execute_next_run_plan.py` validation policy.
2. For official runs, hard checks remain unchanged:
- no skip flags;
- valid freeze file;
- valid non-placeholder DQ input.
3. Command-surface simplification must not weaken audit artifact completeness.
4. Drift-check artifact paths:
- `audit/command_surface/command_surface_check.json`
- `audit/command_surface/command_surface_check.md`
5. Cleanup artifact paths:
- `audit/cleanup/cleanup_report_latest.json`
- `audit/cleanup/cleanup_report_latest.md`

## 4) Deprecation Policy

1. “Internal” does not mean removed.
2. Removal requires:
- no active generator/validator dependency;
- docs and templates migrated;
- one-cycle compatibility observation completed.
